Firewall zero-day exposes admin APIs

Feb 24, 2026 • 3 min read

A remote code execution flaw in a widely deployed firewall lets attackers impersonate the admin interface.

Patches are rolling, but the exploit is already being proofed. Attackers can craft a specially forged packet that appears to come from the vendor update servers; the firewall accepts it and runs commands with SYSTEM privileges.

Expose the firewall only to known management hosts, drop the vendor-specific UDP ports from the edge, and rotate credentials. The update also added stricter TLS validation, so enforce it too.

If you operate a cloud or hybrid environment with these appliances, start a rapid hunt for any new devices added in the last 30 days—those are the ones likely still using the default config that the exploit needs.

Takeaway: Remote management should always be segmented. When a device has a remote-run vulnerability, it only takes one open path to trigger a breach.
